# Strand Identity & Tagline Design

**Date:** 2026-03-04
**Status:** Approved & Implemented

## Core Tagline

**"Stop exploring. Start building."**

## Why It Works

- **For developers:** Stop opening files to understand the codebase. Read `.strand` and know the shape, risk zones, and complexity terrain.
- **For AI agents:** Stop burning 45 tool calls and 70K tokens on navigation. One file read = instant structural awareness.
- **Imperative voice** positions strand as opinionated — "the way you're doing it now is wasteful."
- **Stop/start contrast** implies strand removes work rather than adding it.

## Supporting Line

> Strand encodes your codebase's structure, risk, and complexity into a single version-controlled file — so you and your AI tools understand the whole picture before touching any code.

## Old Tagline Disposition

"Find the right files faster and understand the impact of changes" moved from headline identity to a **features description** in the README body (under "What does strand do?"). It's a *what* statement; the new tagline is a *why* statement.
